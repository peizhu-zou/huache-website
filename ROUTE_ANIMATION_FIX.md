# 首页客户端返回后内容消失：修复说明

问题来自 App Router 的客户端导航：根 `layout.tsx` 不会在页面切换时重新挂载，旧动画控制器只扫描了首次进入页面时存在的元素。返回首页时，缓存或流式插入的 Hero 节点可能错过扫描，因此一直保持透明。

本版采用四层修复：

1. 把 `MotionController` 移入 `app/template.tsx`，每次路由切换都会重挂载。
2. 使用 `MutationObserver` 注册路由切换后新增或流式插入的 `[data-reveal]` 节点。
3. 首屏 Hero 使用 `immediate` 模式，在服务端输出时已经可见，不再依赖 JavaScript 才能显示。
4. CSS 改为“默认可见、JS 渐进增强”，即使脚本初始化失败也不会出现空白内容。

回归测试路径：

- 首页 → 产品中心 → 点击左上角“华车时代” → Hero 与 PowerFlow 应立即显示。
- 首页 → 解决方案 → 返回首页。
- 浏览器后退/前进。
- 直接刷新首页。
